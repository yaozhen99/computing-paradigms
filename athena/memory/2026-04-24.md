## 核心协议要点

### 一、启动流程
1. 读取 `AGENTS.md` 协议
2. 遍历目录 README 构建空间认知
3. 读取最新计划、总结、技能文档
4. 读取模板文件作为格式依据

### 二、文档产生规则（强制）

每个插件/模块必须有的完整文档链：

| 阶段 | 文件位置 | 模板 |
|------|----------|------|
| 需求 | `docs/requirements/[日期]_[简述].md` | requirement.md |
| 计划 | `docs/plans/[日期]_[简述].md` | plan.md |
| 开发笔记 | `docs/notes/[模块名]_开发笔记.md` | dev_note.md |
| 开发总结 | `docs/summaries/[模块名]_开发总结.md` | dev_summary.md |
| 测试需求 | `tests/[模块名]/测试需求.md` | test_req.md |
| 测试计划 | `tests/[模块名]/测试计划.md` | test_plan.md |
| 测试过程 | `tests/[模块名]/测试过程.md` | test_process.md |
| 测试总结 | `tests/[模块名]/测试总结.md` | test_summary.md |
| 审核报告 | `docs/reviews/[模块名]_审核报告.md` | review.md |

### 三、目录驻留文档
- 每个一级、二级子目录必须有 `README.md`
- 修改代码后必须同步更新 README
- 更新记录格式：`## [YYYY-MM-DD HH:MM] 更新说明`

### 四、自我演化机制

**差异分析（强制）**：
- 完成开发总结后，立即对比计划与实际
- 在总结末尾追加 `## 计划与实际差异分析`

**技能沉淀（自动）**：
- 项目特定经验 → `docs/skills/project_specific.md`
- 通用模式 → `docs/skills/general_patterns.md`
- 无新增知识也需追加 `## [日期] 本次任务无新增通用技能`

**协议修正建议**：
- 连续两次相似差异 → 提出协议修正建议
- 建议 → `docs/proposals/protocol_change_[日期].md`
- **严禁直接修改 AGENTS.md**
