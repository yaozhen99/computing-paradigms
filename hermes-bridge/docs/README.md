# docs

**最后更新**：2026-04-23 00:00

## 用途
存放项目所有文档，包括模板、需求、计划、开发笔记、开发总结、审核报告和技能文档。

## 内容概览
| 子目录/文件 | 职责说明 |
|:---|:---|
| `templates/` | 文档格式模板，AI 生成文档时必须严格遵循 |
| `requirements/` | 需求文档，命名格式 `[YYYY-MM-DD]_[简述].md` |
| `plans/` | 计划文档，命名格式 `[YYYY-MM-DD]_[简述].md` |
| `notes/` | 开发笔记，命名格式 `[模块名]_开发笔记.md` |
| `summaries/` | 开发总结，命名格式 `[模块名]_开发总结.md` |
| `reviews/` | 审核报告，命名格式 `[模块名]_审核报告.md` |
| `proposals/` | 协议修正建议，命名格式 `protocol_change_[YYYY-MM-DD].md` |
| `skills/` | 技能文档，包含项目专属技能和通用技能模式 |

## 约定与规范
- 所有文档生成必须遵循 `docs/templates/` 下对应模板的格式要求
- 每个目录下必须存在 `README.md` 作为自描述文件
- 对目录下任何文件进行修改后，必须同步更新该目录的 `README.md`
