# 节点协议（Node Protocol）— 所有岗位的底层规则

## 总则

本协议是小说创作流水线中所有岗位必须遵守的底层规则。无论你是什么角色，以下规则不可违反。

## 1. 文件是唯一真相

- 角色之间**零对话**，所有信息通过文件传递
- 只读取你的岗位被授权的输入文件
- 只写入你的岗位被授权的输出目录
- 不读取其他角色的中间产物（除非协议明确允许）

## 2. 签字协议

完成工作后，在对应的 `_pipes/lock_<stage>.json` 中签字：

```json
{
  "stage": "<stage_name>",
  "status": "completed",
  "signed_by": "<your_role>",
  "signed_at": "<ISO 8601>",
  "output_files": ["<relative/path/to/output>"]
}
```

**签字即承诺**：你确认输出已自检，质量达标。

## 3. 防御性写入

- 先写入 `.tmp` 文件
- 自检内容完整性和格式正确性
- 确认无误后，将 `.tmp` 重命名为正式文件
- 如果自检失败，删除 `.tmp`，**不留下残缺文件**

## 4. 人类圣裁

以下阶段需要人类批准才能继续：

- **worldbuilding** — 世界观设定
- **outline** — 故事大纲

批准方式：人类在 `_system/` 下创建 `approved_<stage>.md` 文件，内容为批准意见。

**没有批准文件 = 没有通过。不得自行推进。**

## 5. 审稿否决

审稿员（editor）有权否决正文。否决时在 `05_review/review_report.md` 中写明：

- 否决原因分类（世界观矛盾 / 大纲偏离 / 章节规划问题 / 正文质量）
- 具体问题描述
- 建议返工阶段

全局 AI 根据否决分类决定回退到哪个阶段。

## 6. 熔断

连续否决达到 3 次（`system_state.json` 中 `veto_count >= max_veto`），流水线暂停，等待人类介入。

人类介入后，重置 `veto_count` 为 0，流水线继续。

## 7. 禁令

- **禁止越权** — 不写不属于你岗位的内容
- **禁止修改上游** — 不修改已完成签字的上游文件
- **禁止跳过** — 不跳过任何必需的工序
- **禁止猜测** — 不猜测其他角色的意图，只看文件
- **禁止半成品** — 不提交未完成的工作，宁可超时也不交残缺品

## 8. 日志

每次执行操作时，在 `_logs/pipeline_execution_log.md` 中追加一条记录：

```
[<ISO 8601>] <role> - <action> - <result>
```
