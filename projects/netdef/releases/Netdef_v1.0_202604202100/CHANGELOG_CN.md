## [1.0.1] - 2026-04-19

### 🔧 GitHub发布准备 - Bug修复与质量改进

#### 修复
- **Wail2Ban state.json文件锁竞争** - 添加基于Mutex的同步和原子文件写入（临时文件+重命名），消除并发访问时的"文件正在使用"错误
- **Wail2Ban多实例防护** - 重写 `wail2ban-manager.bat` 的进程检测，正确返回退出码；BAT层现在阻止重复启动
- **setting.ini中真实IP地址暴露** - 将实际内网IP范围替换为标准示例值（192.168.1.100-192.168.1.200）
- **运行时文件被git跟踪** - 将 `state.json`、`*.json.tmp` 和 `backup_*/` 添加到 `.gitignore`
- **LICENSE中的占位符文本** - 将 `[Your Name/Netdef Contributors]` 替换为 `Netdef Contributors`
- **README.md中的占位符URL** - 将 `YOURNAME` 替换为 `netdef-project`
- **CHANGELOG.md中的占位符** - 将 `[security contact email]` 替换为正确的联系方式
- **Netdef.bat语法错误** - 修复第137行缺少的 `echo` 命令（`ERROR:` → `echo [ERROR]`）
- **HTML GUI语言不一致** - 默认语言从中文改为英文；所有回退文本改为英文；i18n系统保留供未来多语言使用
- **INI解析跨节键冲突** - 在 `lan-config.bat` 和 `outdoor-config.bat` 中用PowerShell节感知解析器替换 `findstr`/`find`
- **临时脚本文件残留** - 添加启动时清理上次运行的 `netdef_*.ps1` 临时文件

#### 变更
- `wail2ban.ps1` Save-State现在使用Mutex + 原子写入（临时文件→重命名），带3次重试机制
- `wail2ban-manager.bat` 进程检测返回退出码供BAT层条件分支
- `.gitignore` 扩展了运行时状态和备份节
- HTML GUI默认语言设为英文（`currentLang = 'en'`，`lang="en"`）

#### 安全
- 仓库中无真实IP地址或网络信息
- Mutex防止并发Wail2Ban实例导致状态文件损坏

## [1.0.0] - 2026-04-15

### 🎉 首次发布 - "Network Definer"

#### 新增
- **核心安全引擎**
  - 零信任入站过滤，支持IP白名单
  - 多配置架构（家庭 / 办公 / 旅行 / 锁定）
  - 公共网络出站端口阻止
  - 通过 `setting.ini` 集中配置

- **动态入侵检测（Wail2Ban）**
  - 自定义Windows兼容Wail2Ban实现
  - 实时安全事件监控（轮询模式，10秒间隔）
  - 逐级递增封禁系统（1h → 5h → 25h → 5d → 90d）
  - 可信IP自动白名单豁免
  - 跨重启持久化状态管理
  - 计划任务集成，支持开机自启

- **用户界面**
  - 交互菜单系统（`netdef.bat`），8个选项
  - 命令行界面支持（`netdef.bat 1-7`）
  - 可配置行数的日志查看器
  - 带确认提示的一键清理
  - 颜色编码状态消息

- **旅行模式特性** ⭐ *亮点功能*
  - 阻止危险出站端口（SMB: 139/445, RDP: 3389等）
  - 异常检测（>100次丢弃/5分钟触发告警）
  - 增强日志记录，用于取证分析
  - 预配置为最大公共WiFi安全

- **文档**
  - 全面的README，包含4个使用场景
  - 内联代码注释（英文）
  - 常见配置示例
  - 故障排除FAQ
  - 安全模型说明（零信任 + 纵深防御）

- **开发者体验**
  - 完整英文本地化（无编码问题）
  - 模块化文件结构（关注点分离）
  - 跨平台兼容命名（小写、连字符）
  - MIT许可，最大程度推广

### 安全
- IP地址和端口范围的输入验证
- 安全的临时文件处理（唯一名称防止竞态条件）
- 增强的错误处理和详细日志
- 执行前验证管理员权限
- 应用防火墙规则前验证白名单
- 缺少配置文件时优雅降级

### AI辅助开发
- 使用 [TRAE SOLO](https://trae.com) AI助手构建
- 自动代码审查发现并修复15个安全问题
- 生成全面的测试用例设计
- 文档辅助提升质量约80%
- 总开发加速：**~4.4x** vs 传统方法
- 开发时间：**72小时**（无AI估计5+周）

### 架构决策
- **Batch + PowerShell混合**：BAT负责编排，PS负责逻辑
- **INI配置**：人类可读，无依赖
- **轮询优于事件订阅**：更可靠，适用于所有Windows版本
- **netsh优于PowerShell cmdlet**：更好的兼容性，更快的执行
- **JSON状态文件**：便于调试，版本控制友好

### 已知限制
- 仅IPv4（IPv6支持计划v2.0）
- 仅Windows（目前无Linux/macOS版本）
- 单机操作（无集中管理）
- 需要手动配置IP范围（自动检测计划中）

## 迁移指南

### 从v0.x（内部构建）到v1.0.0

如果你使用过早期开发构建：

1. **文件重命名**：
   ```
   Netdef.bat → netdef.bat
   1-防火墙局域网配置脚本.bat → lan-config.bat
   2-防火墙外出配置脚本.bat → outdoor-config.bat
   3-防火墙解除规则脚本.bat → cleanup-rules.bat
   ```

2. **配置格式**：`setting.ini` 结构不变（向后兼容）

3. **编码**：所有文件现为UTF-8无BOM（修复乱码问题）

4. **新功能**：旅行模式异常检测、增强日志、英文UI

**无需数据迁移** - 现有 `setting.ini` 可直接使用。

## 安全策略

对于安全漏洞，请**不要**公开提issue。而是：

1. 将详情发送至：security@netdef-project（或开启私密GitHub安全建议）
2. 包含：
   - 漏洞描述
   - 复现步骤
   - 潜在影响
   - 建议修复（如有）

响应时间：确认漏洞后72小时内。

公开披露：修复发布后（或90天未解决）。
